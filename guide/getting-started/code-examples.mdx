## Rate Limiting

The Blitz API enforces a **per-second request limit** on every API key. The exact limit depends on your plan. Exceeding it will result in `429` errors that slow down your entire pipeline.

| Setting                    | Value              |
| :------------------------- | :----------------- |
| **Default rate limit**     | **5 requests/second** (varies by plan) |
| **Enforcement**            | Per Workspace        |
| **Error on exceed**        | `429 Too Many Requests` |
| **Recommended approach**   | Sliding-window rate limiter (see below) |

### Check your rate limit

Your exact rate limit is returned by the [`/v2/account/key-info`](#check-api-key--get-v2accountkey-info) endpoint in the `max_requests_per_seconds` field. **Always read this value** instead of hardcoding it.

```bash
curl -H "x-api-key: YOUR_API_KEY" https://api.blitz-api.ai/v2/account/key-info
```

```json
{
  "valid": true,
  "remaining_credits": 99.5,
  "max_requests_per_seconds": 5,
  ...
}
```

Use the `max_requests_per_seconds` value to initialize your rate limiter. The examples on this page default to `5`, but your plan may allow more.

<Tip>
  **Need a higher rate limit?** You can upgrade your plan for increased throughput. [Book a call](https://cal.com/antoine-blitz-acepmu/blitz-api) or reach out to the team to discuss your needs.
</Tip>

A simple `time.sleep()` between calls is **not reliable** in production. Network latency, retries, and concurrent threads can cause bursts that exceed the limit. Use a **sliding-window rate limiter** that tracks actual request timestamps.

<Warning>
  **Strict requirement**: Failing to implement proper rate limiting will result in `429 Too Many Requests` errors during bulk operations. The rate limiter below is **thread-safe** and handles concurrent requests correctly. Copy it into every project that calls the Blitz API.
</Warning>

### Python Rate Limiter

A factory function that returns a thread-safe `wait()` function. Call `wait()` before every request — it blocks until a slot is available in the sliding window.

```python
import time
import threading
from collections import deque


def create_rate_limiter(max_rps):
    timestamps = deque()
    lock = threading.Lock()

    def wait():
        with lock:
            while True:
                now = time.monotonic()
                while timestamps and timestamps[0] <= now - 1.0:
                    timestamps.popleft()
                if len(timestamps) < max_rps:
                    timestamps.append(time.monotonic())
                    return
                wait_time = timestamps[0] + 1.0 - now + 0.001
                lock.release()
                try:
                    time.sleep(wait_time)
                finally:
                    lock.acquire()

    return wait
```

Usage:

```python
rate_limit = create_rate_limiter(5)

rate_limit()  # blocks until a slot is available
requests.post(...)
```

### Node.js / TypeScript Rate Limiter

A factory function that returns an async `wait()` function. Requests are serialized through a promise chain so concurrent callers never burst past the limit.

```javascript
function createRateLimiter(maxRps) {
  const timestamps = [];
  let pending = Promise.resolve();

  return () => {
    pending = pending.then(
      () =>
        new Promise((resolve) => {
          const poll = () => {
            const now = Date.now();
            while (timestamps.length && timestamps[0] <= now - 1000)
              timestamps.shift();
            if (timestamps.length < maxRps) {
              timestamps.push(Date.now());
              resolve();
            } else {
              setTimeout(poll, timestamps[0] + 1000 - now + 1);
            }
          };
          poll();
        })
    );
    return pending;
  };
}
```

Usage:

```javascript
const rateLimit = createRateLimiter(5);

await rateLimit(); // resolves when a slot is available
fetch(...);
```

## Base Setup

Every request to the Blitz API requires the `x-api-key` header and **must** go through the rate limiter. The client below wires everything together: rate limiting, retries on `429`, and error handling.

**Base URL**: `https://api.blitz-api.ai`

<Note>
  The examples below default to `max_rps=5`. Replace `5` with the `max_requests_per_seconds` value from your [`/v2/account/key-info`](#check-api-key--get-v2accountkey-info) response to match your plan.
</Note>

<CodeGroup>

```python Python
import time
import threading
from collections import deque
import requests

BLITZ_API_KEY = "YOUR_API_KEY"
BLITZ_BASE_URL = "https://api.blitz-api.ai"
HEADERS = {
    "x-api-key": BLITZ_API_KEY,
    "Content-Type": "application/json",
}


def create_rate_limiter(max_rps):
    timestamps = deque()
    lock = threading.Lock()

    def wait():
        with lock:
            while True:
                now = time.monotonic()
                while timestamps and timestamps[0] <= now - 1.0:
                    timestamps.popleft()
                if len(timestamps) < max_rps:
                    timestamps.append(time.monotonic())
                    return
                wait_time = timestamps[0] + 1.0 - now + 0.001
                lock.release()
                try:
                    time.sleep(wait_time)
                finally:
                    lock.acquire()

    return wait


# Default 5 req/s — replace with your plan's max_requests_per_seconds from /v2/account/key-info
rate_limit = create_rate_limiter(5)


def blitz_request(method, path, payload=None):
    url = f"{BLITZ_BASE_URL}{path}"

    for attempt in range(3):
        try:
            rate_limit()

            if method == "GET":
                response = requests.get(url, headers=HEADERS, timeout=60)
            else:
                response = requests.post(url, headers=HEADERS, json=payload, timeout=60)

            if response.status_code == 429:
                print("Rate limited by server, waiting 60s...")
                time.sleep(60)
                continue

            if response.status_code == 402:
                print("Insufficient credits")
                return None

            if response.status_code == 401:
                print("Invalid API key")
                return None

            if response.status_code == 200:
                return response.json()

            time.sleep(2 ** attempt)

        except Exception as e:
            if attempt == 2:
                print(f"API error: {e}")
                return None
            time.sleep(2)

    return None
```

```javascript Node.js
const BLITZ_API_KEY = "YOUR_API_KEY";
const BLITZ_BASE_URL = "https://api.blitz-api.ai";

function createRateLimiter(maxRps) {
  const timestamps = [];
  let pending = Promise.resolve();

  return () => {
    pending = pending.then(
      () =>
        new Promise((resolve) => {
          const poll = () => {
            const now = Date.now();
            while (timestamps.length && timestamps[0] <= now - 1000)
              timestamps.shift();
            if (timestamps.length < maxRps) {
              timestamps.push(Date.now());
              resolve();
            } else {
              setTimeout(poll, timestamps[0] + 1000 - now + 1);
            }
          };
          poll();
        })
    );
    return pending;
  };
}

// Default 5 req/s — replace with your plan's max_requests_per_seconds from /v2/account/key-info
const rateLimit = createRateLimiter(5);

async function blitzRequest(method, path, payload) {
  const headers = {
    "x-api-key": BLITZ_API_KEY,
    "Content-Type": "application/json",
  };

  for (let attempt = 0; attempt < 3; attempt++) {
    try {
      await rateLimit();

      const res = await fetch(`${BLITZ_BASE_URL}${path}`, {
        method,
        headers,
        body: payload ? JSON.stringify(payload) : undefined,
      });

      if (res.status === 429) {
        console.log("Rate limited by server, waiting 60s...");
        await new Promise((r) => setTimeout(r, 60000));
        continue;
      }

      if (res.status === 402) {
        console.error("Insufficient credits");
        return null;
      }

      if (res.status === 401) {
        console.error("Invalid API key");
        return null;
      }

      if (res.ok) {
        return await res.json();
      }

      await new Promise((r) => setTimeout(r, 2 ** attempt * 1000));
    } catch (error) {
      if (attempt === 2) {
        console.error(`API error: ${error.message}`);
        return null;
      }
      await new Promise((r) => setTimeout(r, 2000));
    }
  }

  return null;
}
```

</CodeGroup>

<Warning>
  **Never expose your API key in client-side code** (browser, mobile app). Always call the Blitz API from your backend.
</Warning>

<Note>
  The `rate_limit` / `rateLimit` function is created **once** and shared across all calls. Do not create a new one per request. Every `blitz_request` / `blitzRequest` call in the examples below goes through this rate limiter automatically.
</Note>

**Cursor pagination**: pass the `cursor` from the response into your next request to fetch the next page. When `cursor` is `null`, you've reached the end.

```python
cursor = None
all_companies = []

while True:
    payload = {
        "company": {
            "industry": {"include": ["Software Development"]},
            "hq": {"country_code": ["FR"]},
        },
        "max_results": 50,
    }
    if cursor:
        payload["cursor"] = cursor

    result = blitz_request("POST", "/v2/search/companies", payload)
    all_companies.extend(result["results"])
    cursor = result.get("cursor")

    if not cursor:
        break

print(f"Fetched {len(all_companies)} companies total")
```

## Full Workflow: Search → Enrich → Export

This end-to-end example finds decision-makers at a company, enriches their emails, and outputs a clean list. It includes the full rate limiter — copy this as a starting point for any Blitz API script.

```python
import time
import threading
from collections import deque
import requests
import json

BLITZ_API_KEY = "YOUR_API_KEY"
BLITZ_BASE_URL = "https://api.blitz-api.ai"
HEADERS = {
    "x-api-key": BLITZ_API_KEY,
    "Content-Type": "application/json",
}


def create_rate_limiter(max_rps):
    timestamps = deque()
    lock = threading.Lock()

    def wait():
        with lock:
            while True:
                now = time.monotonic()
                while timestamps and timestamps[0] <= now - 1.0:
                    timestamps.popleft()
                if len(timestamps) < max_rps:
                    timestamps.append(time.monotonic())
                    return
                wait_time = timestamps[0] + 1.0 - now + 0.001
                lock.release()
                try:
                    time.sleep(wait_time)
                finally:
                    lock.acquire()

    return wait


# Replace 5 with your plan's max_requests_per_seconds from /v2/account/key-info
rate_limit = create_rate_limiter(5)


def blitz_request(method, path, payload=None):
    for attempt in range(3):
        try:
            rate_limit()
            if method == "GET":
                r = requests.get(f"{BLITZ_BASE_URL}{path}", headers=HEADERS, timeout=60)
            else:
                r = requests.post(f"{BLITZ_BASE_URL}{path}", headers=HEADERS, json=payload, timeout=60)

            if r.status_code == 429:
                time.sleep(60)
                continue
            if r.status_code in (401, 402):
                return None
            if r.status_code == 200:
                return r.json()
            time.sleep(2 ** attempt)
        except Exception as e:
            if attempt == 2:
                print(f"API error: {e}")
                return None
            time.sleep(2)
    return None


# Step 1: Find decision-makers using waterfall search
search_result = blitz_request("POST", "/v2/search/waterfall-icp-keyword", {
    "company_linkedin_url": "https://www.linkedin.com/company/openai",
    "cascade": [
        {
            "include_title": ["VP Sales", "Head of Sales", "CRO"],
            "exclude_title": ["assistant", "intern"],
            "location": ["WORLD"],
            "include_headline_search": False,
        },
        {
            "include_title": ["Sales Director", "Director of Revenue"],
            "exclude_title": ["junior", "intern"],
            "location": ["WORLD"],
            "include_headline_search": False,
        },
        {
            "include_title": ["CEO", "Founder"],
            "exclude_title": ["intern"],
            "location": ["WORLD"],
            "include_headline_search": False,
        },
    ],
    "max_results": 5,
})

# Step 2: Enrich each person with their work email (rate limiter handles pacing)
leads = []
for person in search_result["results"]:
    p = person["person"]

    email_result = blitz_request("POST", "/v2/enrichment/email", {
        "person_linkedin_url": p["linkedin_url"],
    })

    leads.append({
        "name": p["full_name"],
        "title": p["headline"],
        "linkedin_url": p["linkedin_url"],
        "tier": person["icp"],
        "email": email_result.get("email") if email_result and email_result.get("found") else None,
    })

# Step 3: Output the enriched leads
print(json.dumps(leads, indent=2))
```

## Error Handling Reference

All Blitz API errors follow a consistent pattern. The `blitz_request` client above handles `429`, `401`, and `402` automatically — but you should still check for `None` returns in your code.

| Status | Meaning                | Handled by client?  | Your action                                                    |
| :----- | :--------------------- | :------------------ | :------------------------------------------------------------- |
| `200`  | Success                | Returns JSON        | Process the result.                                            |
| `401`  | Unauthorized           | Returns `None`      | Check your `x-api-key` header — the key is missing or invalid.|
| `402`  | Insufficient credits   | Returns `None`      | Top up credits or wait for the next billing cycle reset.       |
| `422`  | Invalid input          | Retries, then `None`| Check required fields and types against the API reference.     |
| `429`  | Rate limit exceeded    | Waits 60s + retries | Should not happen if using the rate limiter. If it does, the client handles it. |
| `500`  | Internal server error  | Retries with backoff| If persistent after 3 retries, contact support.               |

| `/v2/search/companies`                   | POST   | Find companies by filters          | 1 credit / result   |
