BlitzAPI is designed to work natively with low-code automation platforms. Since we use standard REST hooks, you can integrate us into any workflow engine using a generic **HTTP Request Node**.

Here are the specific configurations for the most popular tools.

## n8n

Use the **HTTP Request** node to interact with BlitzAPI.

<Steps>
  <Step title="Add HTTP Request Node">
    Drag an HTTP Request node into your canvas and configure it as follows:
    
    * **Method**: `POST`
    * **URL**: `https://api.blitz-agency.com/api/enrichment/email` (or any other endpoint)
    * **Authentication**: `Generic Credential Type` -> `Header Auth`
  </Step>
  <Step title="Configure Headers">
    In the "Header Auth" credential (or directly in the Headers section):
    
    * **Name**: `x-api-key`
    * **Value**: `YOUR_API_KEY`
  </Step>
  <Step title="Set JSON Body">
    Set the **Body Content Type** to `JSON`. Map your previous node's output to our parameters:
    
    ```json
    {
      "linkedin_profile_url": "{{ $json.linkedinUrl }}"
    }
    ```
  </Step>
</Steps>

## Make (Integromat)

Use the **HTTP** app and select **"Make a request"**.

<Steps>
  <Step title="Basic Settings">
    * **URL**: `https://api.blitz-agency.com/api/enrichment/email`
    * **Method**: `POST`
  </Step>
  <Step title="Headers">
    Add a new item in the Headers section:
    
    | Key | Value |
    | :--- | :--- |
    | `Content-Type` | `application/json` |
    | `x-api-key` | `YOUR_API_KEY` |
  </Step>
  <Step title="Request Content">
    * **Body type**: `Raw`
    * **Content type**: `JSON (application/json)`
    * **Request content**:
    
    ```json
    {
      "linkedin_profile_url": "{{1.linkedin_url}}"
    }
    ```
  </Step>
</Steps>

## Clay

Clay is perfect for bulk enrichment. You will use the **"Enrich Data"** (HTTP API) integration.

<Steps>
  <Step title="Add Enrichment Column">
    Right-click on your table > **Add Enrichment** > **HTTP API**.
  </Step>
  <Step title="Setup Endpoint">
    * **Method**: `POST`
    * **URL**: `https://api.blitz-agency.com/api/search/waterfall-icp`
  </Step>
  <Step title="Setup Headers">
    * `x-api-key`: `YOUR_API_KEY`
    * `Content-Type`: `application/json`
  </Step>
  <Step title="Setup Body">
    Paste the JSON payload. Use Clay's token selector `{{Column Name}}` to make it dynamic.
    
    ```json
    {
      "company_linkedin_url": "{{Company LinkedIn URL}}",
      "cascade": [
        {
          "include_title": ["CEO", "Founder"],
          "location": ["US"]
        }
      ]
    }
    ```
  </Step>
</Steps>

## Troubleshooting Common Errors

If your integration fails in these tools, check these common issues:

<AccordionGroup>
  <Accordion title="Error 401: Unauthorized">
    **Check your Headers.** Most low-code tools hide headers deep in the settings. Ensure `x-api-key` is exactly spelled as shown, with no extra spaces in the key value.
  </Accordion>

  <Accordion title="Error 400: Bad Request">
    **Check your JSON structure.** Ensure you are sending a raw JSON string. In Make/Integromat, forgetting to select "JSON" as the content type is a common mistake.
  </Accordion>

  <Accordion title="Timeout (Make/n8n)">
    **Increase Timeout Settings.** Real-time signals take 1-3 seconds. Some default timeouts are set to 1000ms. Increase the timeout limit on your HTTP node to **10 seconds** to be safe.
  </Accordion>
</AccordionGroup>
