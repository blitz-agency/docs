## What is the Blitz API MCP Server?

The **Model Context Protocol (MCP)** is an open standard that lets AI assistants connect directly to external data sources instead of relying on their training data.

The **Blitz API MCP Server** gives any compatible AI tool **live access** to the complete Blitz API documentation — every endpoint, parameter, response schema, and code example. When your AI assistant needs to write code that calls Blitz API, it searches the actual documentation instead of guessing.

**MCP Server URL**

```
https://docs.blitz-api.ai/mcp
```

### Why use it?

AI models are trained on static snapshots of the internet. When they generate code for an API, they often hallucinate endpoints that don't exist, use outdated parameter names, or return incorrect response shapes.

With the Blitz API MCP Server connected:

- **No more hallucinated endpoints** — the AI reads the real API reference before generating code.
- **Always up to date** — any time the documentation is updated, your AI tool sees the changes immediately.
- **Accurate request/response schemas** — parameters, types, required fields, and example payloads come straight from the source.
- **Faster development** — skip the back-and-forth of correcting AI-generated code that doesn't match the actual API.

<Tip>
  The AI decides **when** to search the documentation based on your conversation context. You don't need to explicitly ask it to — just mention Blitz API and it will proactively look up the relevant endpoints.
</Tip>

## What content is available?

The MCP server provides a search tool that covers the entire Blitz API documentation:

| Content                  | Description                                                                 |
| :----------------------- | :-------------------------------------------------------------------------- |
| **API endpoints**        | Full reference for every v1 and v2 endpoint — methods, URLs, and parameters |
| **Request schemas**      | Required and optional fields, types, and example payloads                   |
| **Response schemas**     | Complete response structures with field descriptions                        |
| **Waterfall logic**      | How cascade/waterfall search works, ICP scoring, and priority levels        |
| **Code examples**        | cURL commands, JSON payloads, and integration patterns                      |
| **Guides & recipes**     | Enrichment workflows, CRM hygiene, and integration tutorials               |

## How it works

When your AI tool is connected to the Blitz API MCP server:

1. **You ask a question** — for example, *"Write a script that finds the marketing director at a company using Blitz API."*
2. **The AI searches the documentation** — it queries the MCP server for relevant endpoints, parameters, and examples.
3. **You get accurate code** — the response is grounded in the real API specification, not guesswork.

The AI proactively searches when it detects your question relates to Blitz API. You can also explicitly ask it to look something up, like *"Search the Blitz API docs for the email enrichment endpoint."*

<Note>
  Each search returns the most relevant documentation sections. The AI may perform multiple searches during a single conversation to gather all the information it needs.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="The AI isn't using the MCP server">
    Some AI tools require you to explicitly enable the connector for each conversation. In Claude, use the attachments button to select the Blitz API connector. In Cursor, make sure the MCP server shows a green status indicator in your MCP settings.
  </Accordion>

  <Accordion title="Tools not appearing after setup">
    Restart your AI tool after adding the MCP configuration. Most clients need a full restart to detect new MCP servers.
  </Accordion>

  <Accordion title="Getting timeout or connection errors">
    Verify the URL is exactly `https://docs.blitz-api.ai/mcp` — no trailing slash, no extra path segments. If the issue persists, check your network connection and try again.
  </Accordion>

  <Accordion title="Using a client that doesn't support remote MCP">
    Some older clients (like Claude Desktop) don't support remote MCP servers natively. Use the `mcp-remote` wrapper:

    ```json
    {
      "mcpServers": {
        "blitz-api": {
          "command": "npx",
          "args": ["-y", "mcp-remote", "https://docs.blitz-api.ai/mcp"]
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>
