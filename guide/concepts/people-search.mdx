title: "People Search"
description: "Find people across multiple companies using combined company and contact filters. Cursor-based pagination."
---

The **People Search** endpoint (`POST /v2/search/people`) lets you find people across your entire addressable market by combining **company filters** and **people filters** in a single query.

Use it to:
- Build lead lists matching both company ICP _and_ persona criteria in one call
- Search across many companies at once (unlike [Employee Finder](/guide/concepts/employee-finder) which targets one company)
- Filter by company attributes (industry, size, HQ location, type) AND people attributes (job title, seniority, department, location)
- Power ABM list-building workflows with precise targeting

**Cost**: 1 credit per result returned (paid plans: unlimited).

---

## When to Use People Search vs. Other Endpoints

| Use Case | Best Endpoint |
| :------- | :------------ |
| Find people matching company + persona criteria across many companies | **People Search** |
| Find all employees at a _single_ company by role and department | [Employee Finder](/guide/concepts/employee-finder) |
| Find the _single best_ decision-maker at a specific company | [Waterfall ICP](/guide/concepts/waterfall-logic) |
| Find companies only (no people) | [Company Search](/guide/concepts/company-search) |

---

## How It Works

You send a `POST` request with two optional filter objects: `company` and `people`. All filters are combined with **AND** logic — results must match every filter you provide. Within each filter, multiple values use **OR** logic.

The API returns a paginated list of person profiles matching your criteria.

## Request Parameters

### Top-Level Parameters

| Parameter | Type | Required | Default | Description |
| :-------- | :--- | :------- | :------ | :---------- |
| `company` | `object` | No | — | Company filter criteria. All nested fields are optional. |
| `people` | `object` | No | — | People filter criteria. All nested fields are optional. |
| `max_results` | `number` | No | `10` | Results per page. Min: `1`, Max: `50`. |
| `cursor` | `string \| null` | No | `null` | Pagination cursor from a previous response. Pass to get the next page. |

### Company Filter Fields

| Field | Type | Description | Example |
| :---- | :--- | :---------- | :------ |
| `linkedin_url` | `string[]` | Exact LinkedIn URLs of target companies. Limits the search to specific companies. | `["https://www.linkedin.com/company/openai"]` |
| `name.include` | `string[]` | Keywords that must appear in the company name | `["Acme"]` |
| `name.exclude` | `string[]` | Keywords to exclude from the company name | `["Agency"]` |
| `industry.include` | `string[]` | Industry values to include (must use [normalized values](/guide/reference/field-normalization#industry)) | `["IT Services and IT Consulting"]` |
| `industry.exclude` | `string[]` | Industry values to exclude | `["Nonprofit"]` |
| `type.include` | `string[]` | Company type to include (see [Company Type values](/guide/reference/field-normalization#company-type)) | `["Privately Held", "Public Company"]` |
| `type.exclude` | `string[]` | Company type to exclude | `["Nonprofit", "Government Agency"]` |
| `employee_range` | `string[]` | Employee count ranges (see [Employee Range values](/guide/reference/field-normalization#employee-range)) | `["51-200", "201-500"]` |
| `employee_count.min` | `number` | Minimum exact employee count (0–980,000) | `50` |
| `employee_count.max` | `number` | Maximum exact employee count (0–1,000,000) | `500` |
| `min_linkedin_followers` | `number` | Minimum LinkedIn followers (0–10,000,000) | `1000` |
| `keywords.include` | `string[]` | Keywords that must appear in the company description | `["SaaS", "B2B"]` |
| `keywords.exclude` | `string[]` | Keywords to exclude from the company description | `["agency"]` |
| `founded_year.min` | `number` | Minimum founding year | `2015` |
| `founded_year.max` | `number` | Maximum founding year | `2024` |
| `hq.city.include` | `string[]` | HQ city keywords to include | `["San Francisco", "New York"]` |
| `hq.city.exclude` | `string[]` | HQ city keywords to exclude | `["Houston"]` |
| `hq.country_code` | `string[]` | HQ country codes (2-letter ISO) | `["US", "GB"]` |
| `hq.continent` | `string[]` | HQ continent (see [Continents](/guide/reference/field-normalization#continents)) | `["North America"]` |
| `hq.sales_region` | `string[]` | HQ sales region (see [Sales Regions](/guide/reference/field-normalization#sales-regions)) | `["NORAM"]` |

<Tip>
  **`employee_range` vs `employee_count`**: Use `employee_range` for standard bucket filters (e.g., `"51-200"`). Use `employee_count` with `min`/`max` for precise numeric ranges (e.g., 100–300 employees exactly).
</Tip>

### People Filter Fields

| Field | Type | Default | Description | Example |
| :---- | :--- | :------ | :---------- | :------ |
| `job_title.include` | `string[]` | `[]` | Job title keywords to include | `["VP Sales", "Head of Sales"]` |
| `job_title.exclude` | `string[]` | `[]` | Job title keywords to exclude | `["Assistant", "Intern"]` |
| `include_linkedin_headline` | `boolean` | `false` | Whether to also search the LinkedIn headline (not just job title) | `true` |
| `job_function` | `string[]` | `[]` | Department / job function (see [Job Functions](/guide/reference/field-normalization#job-functions)) | `["Sales & Business Development"]` |
| `job_level` | `string[]` | — | Seniority level (see [Job Levels](/guide/reference/field-normalization#job-levels)) | `["C-Team", "VP", "Director"]` |
| `min_connections` | `number` | `0` | Minimum LinkedIn connections (0–500). Filters out inactive profiles. | `100` |
| `location.city` | `string[]` | `[]` | Person's city | `["Paris", "London"]` |
| `location.country` | `string[]` | `[]` | Person's country (2-letter ISO codes) | `["FR", "GB"]` |
| `location.continent` | `string[]` | `[]` | Person's continent (see [Continents](/guide/reference/field-normalization#continents)) | `["Europe"]` |
| `location.sales_region` | `string[]` | `[]` | Person's sales region (see [Sales Regions](/guide/reference/field-normalization#sales-regions)) | `["EMEA"]` |

<Warning>
  **Field naming note:** The person country filter in People Search is **`location.country`** (not `country_code` as in Employee Finder). The values are the same ISO 2-letter codes. All enum values are **case-sensitive**. Copy-paste from the [Field Normalization reference](/guide/reference/field-normalization).
</Warning>

## Example Request

Find VPs and Directors in Sales at SaaS companies (51–500 employees) headquartered in the US:

<CodeGroup>

```bash cURL
curl -X POST "https://api.blitz-api.ai/v2/search/people" \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "company": {
      "keywords": { "include": ["SaaS"] },
      "employee_range": ["51-200", "201-500"],
      "hq": { "country_code": ["US"] }
    },
    "people": {
      "job_title": { "include": ["Sales"] },
      "job_level": ["VP", "Director"],
      "job_function": ["Sales & Business Development"]
    },
    "max_results": 10
  }'
```

```javascript Node.js
const result = await blitzRequest("POST", "/v2/search/people", {
  company: {
    keywords: { include: ["SaaS"] },
    employee_range: ["51-200", "201-500"],
    hq: { country_code: ["US"] },
  },
  people: {
    job_title: { include: ["Sales"] },
    job_level: ["VP", "Director"],
    job_function: ["Sales & Business Development"],
  },
  max_results: 10,
});

for (const person of result.results) {
  console.log(`${person.full_name} — ${person.headline}`);
  console.log(`  LinkedIn: ${person.linkedin_url}`);
  console.log(`  Location: ${person.location.city}, ${person.location.country_code}`);
}

// Paginate
if (result.cursor) {
  const nextPage = await blitzRequest("POST", "/v2/search/people", {
    company: { keywords: { include: ["SaaS"] }, /* ... same filters ... */ },
    people: { job_level: ["VP", "Director"], /* ... same filters ... */ },
    max_results: 10,
    cursor: result.cursor,
  });
}
```

```python Python
result = blitz_request("POST", "/v2/search/people", {
    "company": {
        "keywords": {"include": ["SaaS"]},
        "employee_range": ["51-200", "201-500"],
        "hq": {"country_code": ["US"]},
    },
    "people": {
        "job_title": {"include": ["Sales"]},
        "job_level": ["VP", "Director"],
        "job_function": ["Sales & Business Development"],
    },
    "max_results": 10,
})

for person in result["results"]:
    print(f"{person['full_name']} — {person['headline']}")
    print(f"  LinkedIn: {person['linkedin_url']}")
    print(f"  Location: {person['location']['city']}, {person['location']['country_code']}")

# Paginate
if result["cursor"]:
    next_page = blitz_request("POST", "/v2/search/people", {
        "company": {"keywords": {"include": ["SaaS"]}, ...},
        "people": {"job_level": ["VP", "Director"], ...},
        "max_results": 10,
        "cursor": result["cursor"],
    })
```

</CodeGroup>

### Advanced Example: Targeting Specific Companies + Excluding Industries

Find engineering managers at specific companies, excluding nonprofits:

```javascript
const result = await blitzRequest("POST", "/v2/search/people", {
  company: {
    linkedin_url: [
      "https://www.linkedin.com/company/openai",
      "https://www.linkedin.com/company/anthropic",
      "https://www.linkedin.com/company/google",
    ],
    industry: { exclude: ["Non-profit Organization Management"] },
  },
  people: {
    job_level: ["Manager", "Director"],
    job_function: ["Engineering"],
    min_connections: 100,
  },
  max_results: 50,
});
```

## Pagination

People Search uses **cursor-based pagination** (same as [Company Search](/guide/concepts/company-search)). Each response includes a `cursor` field. Pass it in the next request to get the following page. When `cursor` is `null`, you've reached the last page.

```javascript
// Paginate through all results
let cursor = null;
let allResults = [];

do {
  const result = await blitzRequest("POST", "/v2/search/people", {
    company: {
      keywords: { include: ["SaaS"] },
      employee_range: ["51-200", "201-500"],
    },
    people: {
      job_level: ["VP", "Director"],
      job_function: ["Sales & Business Development"],
    },
    max_results: 50,
    cursor: cursor,
  });

  allResults = allResults.concat(result.results);
  cursor = result.cursor;

  console.log(`Fetched ${result.results_length} results (total: ${allResults.length})`);
} while (cursor !== null);
```

## Combining with Enrichment

People Search returns LinkedIn profile URLs but **not** emails or phone numbers. Chain with enrichment endpoints to complete the data:

<Steps>
  <Step title="Search people">
    Use People Search to find matching contacts. Each result includes a `linkedin_url`.
  </Step>
  <Step title="Enrich emails">
    Pass each `linkedin_url` to `POST /v2/enrichment/email` to get verified work emails.
  </Step>
  <Step title="Enrich phones (US only)">
    Pass each `linkedin_url` to `POST /v2/enrichment/phone` for direct mobile numbers.
  </Step>
  <Step title="Validate and sync">
    Validate emails via `POST /v2/utilities/email/validate`, then sync clean data to your CRM.
  </Step>
</Steps>

## Filter Logic Summary

- **Between filters** (e.g., `company.industry` AND `people.job_level`): **AND** — results must match all filters.
- **Within a filter** (e.g., `job_level: ["VP", "Director"]`): **OR** — matches VP or Director.
- **Include/Exclude** (e.g., `industry.include` + `industry.exclude`): Include is applied first, then exclude removes matches.

## Comparison: Search Endpoints at a Glance

| Feature | People Search | Employee Finder | Company Search | Waterfall ICP |
| :------ | :------------ | :-------------- | :------------- | :------------ |
| **Endpoint** | `/v2/search/people` | `/v2/search/employee-finder` | `/v2/search/companies` | `/v2/search/waterfall-icp-keyword` |
| **Scope** | People across many companies | People at one company | Companies only | Best contact at one company |
| **Company filters** | 12 fields | 1 (`company_linkedin_url`) | 5 fields | 1 (`company_linkedin_url`) |
| **People filters** | 6 fields | 5 fields | N/A | Cascade tiers |
| **Pagination** | Cursor-based | Page-based | Cursor-based | N/A (max_results only) |
| **Response nesting** | `results[]` (direct) | `results[]` (direct) | `results[]` (direct) | `results[].person` (nested) |
| **Trial cost** | 1 credit / result | 1 credit / result | 1 credit / result | 1 credit / result |
